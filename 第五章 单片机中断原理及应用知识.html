第五章 单片机中断原理及应用

高等教育出版社 · 魏寿明主编《单片机技术与应用》

学习目标

  • 理解中断的基本概念和作用
  • 掌握8051单片机中断系统的组成和工作原理
  • 学会中断初始化的方法和步骤
  • 理解堆栈在中断处理中的作用
  • 能够编写简单的中断控制程序
  • 掌握中断优先级的设置和处理原则

1 中断概述

1. 中断的定义

中断是指CPU在执行程序的过程中，遇到突发事件时，暂停当前程序的执行，转去处理突发事件，处理完毕后，再回到原来被暂停的程序继续执行的过程。

中断机制是单片机系统中非常重要的组成部分，它使得单片机能够实时响应外部事件，提高了系统的处理能力和灵活性。

2. 中断的作用

  • 提高CPU的工作效率：CPU可以在没有中断请求时执行主程序，有中断请求时才去处理中断，避免了CPU的空转等待。
  • 实时处理：能够及时响应外部事件，满足实时控制系统的要求。
  • 故障处理：可以及时处理系统中的故障，如电源掉电、数据错误等。
  • 实现多任务：通过中断机制，可以实现单片机的多任务处理。

中断的基本概念

中断响应

CPU暂停当前程序，转去执行中断服务程序

中断服务程序

处理中断事件的程序代码

中断返回

中断服务程序执行完毕后返回主程序

断点保护

保存主程序的执行位置，以便中断返回后继续执行

3. 中断系统的组成

硬件组成

  • 中断源：产生中断请求的设备或事件
  • 中断控制器：管理和处理中断请求
  • CPU：执行中断响应和处理

软件组成

  • 中断服务程序：处理中断事件的程序
  • 中断初始化程序：设置中断参数
  • 中断向量表：存储中断服务程序的入口地址

4. 8051单片机的中断系统

8051单片机有5个中断源（基本型），分别是：外部中断0（INT0）、定时器/计数器0溢出中断（TF0）、外部中断1（INT1）、定时器/计数器1溢出中断（TF1）和串行口中断（RI/TI）。

中断源 中断请求标志 中断向量地址 默认优先级
外部中断0（INT0） IE0 0003H 最高
定时器/计数器0溢出中断（TF0） TF0 000BH 次高
外部中断1（INT1） IE1 0013H 中
定时器/计数器1溢出中断（TF1） TF1 001BH 次低
串行口中断（RI/TI） RI/TI 0023H 最低

知识扩展

8052单片机在8051的基础上增加了一个定时器/计数器2中断，中断请求标志为TF2/EXF2，中断向量地址为002BH。

2 中断产生，响应与撤除

1. 中断源分类

外部中断

由外部设备产生的中断请求，用于响应外部事件。

  • 外部中断0（INT0）：由P3.2引脚输入
  • 外部中断1（INT1）：由P3.3引脚输入
  • 触发方式可设置为电平触发或边沿触发

内部中断

由单片机内部事件产生的中断请求，用于处理内部事务。

  • 定时器/计数器溢出中断：TF0、TF1
  • 串行口发送/接收中断：RI/TI
  • 其他内部中断（如8052的定时器2中断）

2. 中断请求标志

每个中断源都有一个中断请求标志位，用于标志该中断源是否有中断请求。这些标志位通常位于特殊功能寄存器中。

中断源 请求标志 所在寄存器 标志置位条件
外部中断0（INT0） IE0 TCON 外部信号触发（电平或边沿）
定时器/计数器0溢出中断 TF0 TCON 定时器/计数器0溢出
外部中断1（INT1） IE1 TCON 外部信号触发（电平或边沿）
定时器/计数器1溢出中断 TF1 TCON 定时器/计数器1溢出
串行口中断 RI/TI SCON 串行口接收完成(RI)/发送完成(TI)

3. 中断响应条件

必要条件

  1. 1 中断源有中断请求（相应的中断请求标志置1）
  2. 2 此中断源的中断允许位为1（IE寄存器相应位为1）
  3. 3 CPU开中断（总中断允许位EA=1）

附加条件

  1. 1 无同级或更高级中断正在被服务
  2. 2 当前执行的指令不是RETI或访问IE、IP的指令
  3. 3 若为上述指令，则需等待该指令执行完毕且再执行一条指令后

4. 中断响应过程

1. 检测中断请求

CPU在每个机器周期的S5P2时刻检测各中断源的中断请求标志位。

2. 判断响应条件

检查是否满足中断响应的所有条件，若满足则响应中断，否则忽略。

3. 保护断点

将当前PC值（断点地址）压入堆栈，以便中断返回后能继续执行主程序。

4. 关中断（部分情况）

对于外部中断0和外部中断1，响应后会自动关闭同级中断；其他中断不会自动关闭。

5. 转去执行中断服务程序

根据中断源的向量地址，将对应的中断服务程序入口地址装入PC，开始执行中断服务程序。

5. 中断撤除

中断撤除是指清除中断请求标志，以免重复响应同一中断请求。不同中断源的撤除方式不同：

自动撤除

  • 定时器溢出中断：CPU响应中断后自动清除TF0/TF1
  • 边沿触发的外部中断：CPU响应中断后自动清除IE0/IE1

手动撤除

  • 电平触发的外部中断：需外部电路配合清除
  • 串行口中断：需在中断服务程序中手动清除RI/TI

注意事项

对于电平触发的外部中断，如果中断请求信号不能及时撤除，可能会导致中断服务程序执行完毕后再次响应同一中断请求，造成重复中断。

3 中断初始化

1. 中断初始化的步骤

1

设置中断请求触发方式

对于外部中断，需要设置其触发方式：电平触发或边沿触发。这通过TCON寄存器中的IT0和IT1位来实现。 

IT0 = 0;  // 外部中断0为电平触发（低电平有效）
IT0 = 1;  // 外部中断0为边沿触发（下降沿有效）
IT1 = 0;  // 外部中断1为电平触发（低电平有效）
IT1 = 1;  // 外部中断1为边沿触发（下降沿有效）
2

设置中断优先级

根据需要设置各中断源的优先级，通过IP寄存器实现。8051支持两级优先级：高优先级和低优先级。 

PX0 = 1;  // 外部中断0设置为高优先级
PT0 = 0;  // 定时器0中断设置为低优先级
PX1 = 1;  // 外部中断1设置为高优先级
PT1 = 0;  // 定时器1中断设置为低优先级
PS  = 1;  // 串行口中断设置为高优先级
3

开中断

打开总中断允许位EA，并允许相应中断源的中断请求。这通过IE寄存器实现。 

EA = 1;   // 开总中断
EX0 = 1;  // 允许外部中断0中断
ET0 = 1;  // 允许定时器0中断
EX1 = 1;  // 允许外部中断1中断
ET1 = 1;  // 允许定时器1中断
ES  = 1;  // 允许串行口中断

2. 中断控制寄存器

IE寄存器（中断允许寄存器）

用于控制各中断源的中断允许或禁止。

位 D7 D6 D5 D4 D3 D2 D1 D0
位名称 EA - ET2 ES ET1 EX1 ET0 EX0
功能 总中断允许 - 定时器2允许 串行口允许 定时器1允许 外部中断1允许 定时器0允许 外部中断0允许

IP寄存器（中断优先级寄存器）

用于设置各中断源的优先级。

位 D7 D6 D5 D4 D3 D2 D1 D0
位名称 - - PT2 PS PT1 PX1 PT0 PX0
功能 - - 定时器2优先级 串行口优先级 定时器1优先级 外部中断1优先级 定时器0优先级 外部中断0优先级

3. 中断初始化实例

下面是一个外部中断0的初始化示例，设置为边沿触发方式，高优先级： 

void init_interrupt0(void) {
    IT0 = 1;    // 设置外部中断0为边沿触发方式
    PX0 = 1;    // 设置外部中断0为高优先级
    EX0 = 1;    // 使能外部中断0
    EA = 1;     // 开总中断
}

初始化注意事项

  • 初始化顺序一般为：先设置触发方式，再设置优先级，最后开中断
  • 高优先级中断可以打断低优先级中断，形成中断嵌套
  • 同级中断按固定顺序查询响应：INT0 → TF0 → INT1 → TF1 → RI/TI

4 堆栈

1. 堆栈的基本概念

堆栈是单片机内存中的一个特殊区域，用于临时存放数据和地址。它遵循"后进先出"(LIFO)的原则，类似于一摞盘子，最后放上去的盘子总是最先被拿走。

堆栈的主要作用

  • 断点保护

    在中断或子程序调用时保存返回地址

  • 现场保护

    保存寄存器内容，确保中断返回后原程序能继续正确执行

  • 参数传递

    在子程序调用时传递参数和返回结果

堆栈操作指令

  • PUSH

    将数据压入堆栈，SP指针加1

  • POP

    从堆栈弹出数据，SP指针减1

  • MOV SP, #data

    设置堆栈指针初值

2. 堆栈指针(SP)

堆栈指针(SP)是一个8位寄存器，用于指示堆栈的栈顶位置。8051单片机复位后，SP的初始值为07H，即堆栈从08H地址开始。

3. 堆栈在中断处理中的作用

在中断处理过程中，堆栈主要用于保存断点地址和现场数据：

断点保护

当CPU响应中断时，会自动将当前PC值（断点地址）压入堆栈，以便中断返回时能回到原程序继续执行。

例如，当外部中断0发生时，CPU会自动执行以下操作： 

PUSH PC_low    ;保存PC低8位
PUSH PC_high   ;保存PC高8位
PC = 0003H     ;跳转到外部中断0的中断服务程序入口

现场保护

在中断服务程序中，通常需要保护现场，即将中断服务程序中要使用的寄存器内容压入堆栈，以免破坏主程序的运行环境。

例如，一个典型的中断服务程序结构： 

INT0_ISR:
    PUSH ACC    ;保护累加器
    PUSH PSW    ;保护程序状态字
    ;中断服务程序主体
    POP PSW     ;恢复程序状态字
    POP ACC     ;恢复累加器
    RETI        ;中断返回

4. 堆栈的设置与使用注意事项

  • 堆栈空间设置：8051复位后SP初值为07H，堆栈从08H开始。由于08H-1FH是工作寄存器区，20H-2FH是位寻址区，建议将SP值初始化为60H或更大值，避免堆栈覆盖这些区域。
  • 堆栈深度：堆栈空间有限，避免过深的中断嵌套或子程序调用，防止堆栈溢出。
  • PUSH与POP配对：在中断服务程序中，PUSH和POP指令必须配对使用，确保堆栈平衡。
  • 中断嵌套：高优先级中断可以打断低优先级中断，此时堆栈需要保存多个断点和现场数据。

5 中断控制案例

1. 外部中断控制LED

本案例使用外部中断0(INT0)控制LED的亮灭，当外部按键按下时，触发中断，LED状态翻转。

硬件连接

  • LED正极 → P1.0
  • LED负极 → GND
  • 按键一端 → P3.2(INT0)
  • 按键另一端 → GND
  • P3.2需接10KΩ上拉电阻到VCC

程序代码

ORG 0000H
    LJMP MAIN     ;跳转到主程序

ORG 0003H         ;外部中断0入口地址
    LJMP INT0_ISR ;跳转到中断服务程序

ORG 0030H         ;主程序起始地址
MAIN:
    SETB IT0      ;设置外部中断0为边沿触发
    SETB EX0      ;使能外部中断0
    SETB EA       ;开总中断
    
    CLR P1.0      ;初始状态LED熄灭
    
MAIN_LOOP:
    SJMP MAIN_LOOP ;主程序循环

;外部中断0服务程序
ORG 0100H
INT0_ISR:
    PUSH ACC      ;保护累加器
    PUSH PSW      ;保护程序状态字
    
    CPL P1.0      ;翻转LED状态
    
    POP PSW       ;恢复程序状态字
    POP ACC       ;恢复累加器
    RETI          ;中断返回
    END

2. 定时器中断实现LED闪烁

本案例使用定时器0中断实现LED以固定频率闪烁。

硬件连接

  • LED正极 → P1.0
  • LED负极 → GND

程序代码

ORG 0000H
    LJMP MAIN     ;跳转到主程序

ORG 000BH         ;定时器0中断入口地址
    LJMP TIMER0_ISR ;跳转到中断服务程序

ORG 0030H         ;主程序起始地址
MAIN:
    MOV TMOD, #01H ;设置定时器0为模式1(16位定时器)
    MOV TH0, #0FCH ;设置定时器初值
    MOV TL0, #66H  ;定时1ms(晶振12MHz)
    
    SETB ET0      ;使能定时器0中断
    SETB EA       ;开总中断
    SETB TR0      ;启动定时器0
    
    CLR P1.0      ;初始状态LED熄灭
    
MAIN_LOOP:
    SJMP MAIN_LOOP ;主程序循环

;定时器0中断服务程序
ORG 0100H
TIMER0_ISR:
    PUSH ACC      ;保护累加器
    PUSH PSW      ;保护程序状态字
    
    MOV TH0, #0FCH ;重新加载初值
    MOV TL0, #66H
    
    ;使用R7作为计数器
    DJNZ R7, TIMER_EXIT
    MOV R7, #195  ;500ms = 500次*1ms
    
    CPL P1.0      ;翻转LED状态
    
TIMER_EXIT:
    POP PSW       ;恢复程序状态字
    POP ACC       ;恢复累加器
    RETI          ;中断返回
    END

3. 中断嵌套示例

本案例演示中断嵌套的使用，外部中断0(高优先级)可以打断外部中断1(低优先级)的服务程序。

程序代码

ORG 0000H
    LJMP MAIN     ;跳转到主程序

ORG 0003H         ;外部中断0入口地址
    LJMP INT0_ISR ;跳转到中断服务程序

ORG 0013H         ;外部中断1入口地址
    LJMP INT1_ISR ;跳转到中断服务程序

ORG 0030H         ;主程序起始地址
MAIN:
    SETB IT0      ;设置外部中断0为边沿触发
    SETB IT1      ;设置外部中断1为边沿触发
    
    SETB PX0      ;设置外部中断0为高优先级
    CLR PX1       ;设置外部中断1为低优先级
    
    SETB EX0      ;使能外部中断0
    SETB EX1      ;使能外部中断1
    SETB EA       ;开总中断
    
    CLR P1.0      ;初始状态LED0熄灭
    CLR P1.1      ;初始状态LED1熄灭
    
MAIN_LOOP:
    SJMP MAIN_LOOP ;主程序循环

;外部中断0服务程序(高优先级)
ORG 0100H
INT0_ISR:
    PUSH ACC      ;保护累加器
    PUSH PSW      ;保护程序状态字
    
    SETB P1.0     ;点亮LED0
    
    ;延时一段时间
    MOV R2, #200
DELAY0_1:
    MOV R3, #250
DELAY0_2:
    DJNZ R3, DELAY0_2
    DJNZ R2, DELAY0_1
    
    CLR P1.0      ;熄灭LED0
    
    POP PSW       ;恢复程序状态字
    POP ACC       ;恢复累加器
    RETI          ;中断返回

;外部中断1服务程序(低优先级)
ORG 0200H
INT1_ISR:
    PUSH ACC      ;保护累加器
    PUSH PSW      ;保护程序状态字
    
    SETB P1.1     ;点亮LED1
    
    ;延时一段时间，期间如果有INT0请求，会被打断
    MOV R4, #200
DELAY1_1:
    MOV R5, #250
DELAY1_2:
    DJNZ R5, DELAY1_2
    DJNZ R4, DELAY1_1
    
    CLR P1.1      ;熄灭LED1
    
    POP PSW       ;恢复程序状态字
    POP ACC       ;恢复累加器
    RETI          ;中断返回
    END

4. 中断编程注意事项

常见错误

  • 忘记开总中断(EA=1)
  • 中断服务程序中没有保护和恢复现场
  • 中断服务程序执行时间过长
  • 使用了与中断向量地址冲突的代码

编程建议

  • 中断服务程序应尽量简短
  • 使用寄存器组切换保护现场
  • 避免在中断服务程序中使用延时函数
  • 对共享资源访问时应进行保护

6 本章总结

1. 重点回顾

核心概念

  • 中断：CPU暂停当前程序，转去处理突发事件的机制
  • 中断源：引起中断的原因或设备
  • 中断优先级：多个中断同时发生时的处理顺序
  • 中断向量表：存储中断服务程序入口地址的表格
  • 堆栈：用于保存断点和现场数据的内存区域

关键寄存器

  • IE：中断允许寄存器，控制各中断源的允许/禁止
  • IP：中断优先级寄存器，设置中断优先级
  • TCON：定时器控制寄存器，设置外部中断触发方式
  • SCON：串行口控制寄存器，控制串行口中断
  • SP：堆栈指针，指示堆栈栈顶位置

2. 中断编程要点

中断响应过程

  1. 中断源发出中断请求
  2. CPU检测到中断请求，且中断允许
  3. CPU保护断点（将PC压入堆栈）
  4. 根据中断源，将相应的中断向量地址装入PC
  5. 执行中断服务程序
  6. 执行RETI指令，恢复断点（从堆栈弹出PC）
  7. 继续执行原程序

中断服务程序结构

中断服务程序:
    PUSH ACC    ;保护累加器
    PUSH PSW    ;保护程序状态字
    ;设置工作寄存器组(可选)
    MOV R0, #数据 ;中断服务程序主体
    ;...
    POP PSW     ;恢复程序状态字
    POP ACC     ;恢复累加器
